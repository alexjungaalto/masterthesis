# CLAUDE.md

## Project Overview

This is a **public GitHub repository** providing guidance to master thesis students supervised by Alex Jung (Associate Professor for Machine Learning, Aalto University). The audience is current and prospective supervisees.

## Repository Structure

- `README.md` — Main guide: supervisor/student expectations, getting started, timeline, writing conventions, evaluation process
- `topics.csv` — **Source of truth** for available thesis topic proposals. Hand-edit this file.
- `compile_topics.py` — Python script to compile and filter the topic catalog
- `Topics.md` — **Auto-generated** from `topics.csv` via `compile_topics.py` (do not edit directly)
- `theses.csv` — **Source of truth** for all supervised master theses (Aalto + TU Wien). Hand-edit this file.
- `compile_theses.py` — Python script to compile, filter, and export the thesis list
- `theses.md` — **Auto-generated** from `theses.csv` via `compile_theses.py` (do not edit directly)
- `theses.tex` — **Auto-generated** from `theses.csv` via `compile_theses.py` (do not edit directly)
- `material/` — PDFs and templates (grade characterization, self-assessment form, peer-review forms, current thesis list PDF)
- `material/creategraphtex.py` — Utility to extract LaTeX document structure as a graph
- `mkdocs.yml`, `build_site.sh`, `requirements-docs.txt` — Static website (ml-theses.org) build config; see `DEPLOY.md`

## Key Conventions

- **Audience**: All content is student-facing. Write clearly, accessibly, and actionably.
- **Terminology**: Use the [Aalto Dictionary of ML](https://aaltodictionaryofml.github.io/) for ML terms.
- **References**: Format citations per IEEE guidelines.
- **AI disclosure**: AI tool usage disclosures go in a dedicated statement, NOT in the Methods section (which is reserved for research methods).
- **No sensitive content**: This is a public repo. Do not add internal notes, credentials, or draft-quality content.

## Working with the Thesis List

- **`theses.csv` is the only file to hand-edit.** Columns: `number, author, title, university, status, date, industry, url, recording`.
- `recording` is an optional YouTube link to the thesis presentation (from Alex Jung's channel). Leave blank when no recording exists; verify the URL points to the correct student's talk before adding it.
- `theses.md` and `theses.tex` are **build artifacts** generated by `compile_theses.py`. Never edit them directly — changes will be overwritten on the next run.
- After any edit to `theses.csv`, regenerate both outputs:
  ```
  python compile_theses.py --markdown --tex
  ```
- Run `python compile_theses.py --stats` for summary statistics.
- When adding new theses, append to `theses.csv` and re-run the script.
- Ground truth for author names and titles is the `citation_author` / `citation_title` metadata on the linked Aaltodoc record. Use proper diacritics (e.g. `Hämmäinen`, not `Hamminen`).
- The PDF `material/MasterThesisSupervisedCurrent.pdf` is the original source but may lag behind the CSV.

## Working with Topic Proposals

- **`topics.csv` is the only file to hand-edit.** Columns: `id, title, area, data_source, difficulty, status, description, url`.
- `Topics.md` is a **build artifact** generated by `compile_topics.py`. Never edit it directly — changes will be overwritten on the next run.
- After any edit to `topics.csv`, regenerate the catalog:
  ```
  python compile_topics.py --markdown
  ```
- `status` is `open` or `taken`. The published catalog lists only `open` topics by default; use `--all` to include taken ones.
- Each topic is one short CSV row (no per-topic long-form pages). Keep `description` to a concise paragraph.

## Style Notes

- Do not use emojis in README.md or the topic catalog.
- Keep markdown well-structured with a table of contents for long documents.
- Link to Aalto's official policies (responsible AI use, academic appeals) rather than paraphrasing them extensively.
